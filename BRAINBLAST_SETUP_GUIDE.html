⏱️ Be Patient!
The first time Unity opens any project, it generates a Library/ folder. This is a cache of compiled scripts, imported assets, and metadata. This process can take 5–15 minutes depending on your hardware. You'll see a progress bar at the bottom of the Unity splash that says "Importing Assets." Do not close Unity during this process.

2.4 Safe Mode

If Unity detects compilation errors, it may open in Safe Mode — a stripped-down editor with an orange banner at the top saying "Safe Mode: Script compilation errors."

💡 Don't Panic
Safe Mode usually means a dependency is missing. The most common cause with BrainBlast is the TextMeshPro (TMP) package. To fix:
  1. Go to Window → Package Manager.
  2. Search for "TextMeshPro" — it should already be installed. If prompted to "Import TMP Essentials," click it.
  3. If errors persist, check the Console (Window → Console) for specific error messages. Usually closing and reopening Unity resolves transient import issues.

3.3 Architecture Overview

BrainBlast uses a single-scene architecture with two scenes:

  1. SplashScreen — A minimal scene that shows the logo and fades into scene 2.
  2. Gameplay — Contains ALL game panels (MainMenu, LevelSelect, Gameplay, Victory, Settings, Shop, Pause) managed by UIManager. Only one panel is visible at a time.

How it works:

  • GameInitializer runs on Awake and creates all singleton managers if they don't exist.
  • UIManager activates/deactivates panel GameObjects to switch screens — no scene loading needed for navigation.
  • All managers use the Singleton pattern (ManagerName.Instance) and persist via DontDestroyOnLoad or live within the Gameplay scene.
  • Level data is loaded from Resources/LevelData/levels.json at runtime.
💡 Why Single-Scene?
Keeping everything in one scene means faster transitions (no loading screens between menus), simpler state management, and fewer bugs from cross-scene references. The UIManager simply toggles panel visibility.

Chapter 4 — Building the Scenes

The project ships with the scripts but the scenes need to be generated by the SceneBuilder tool. This is the most important step!

4.1 Build All Scenes

  1. In the Unity Editor, look at the top menu bar. You'll see the standard menus: File, Edit, Assets, GameObject, Component, Window, Help — and a custom menu called BrainBlast.
  2. Click BrainBlast → Build All Scenes.
  3. Unity will run the SceneBuilder script. You'll see progress in the Console panel. The builder creates:
    • SplashScreen.unity — Logo image, background, fade-to-black overlay, GameInitializer object.
    • Gameplay.unity — Main Canvas with all UI panels:
      • MainMenuPanel — Title, Play button, Settings button, Daily Puzzle button
      • LevelSelectPanel — World tabs, scrollable level grid with 40 level buttons per world
      • GameplayPanel — Tile grid area, score display, move counter, pause button, hint button
      • VictoryPanel — Star rating display, score, "Next Level" and "Replay" buttons
      • SettingsPanel — Sound/music toggles, theme toggle, about info
      • ShopPanel — Coin packs, ad removal, hint refills
      • PausePanel — Resume, restart, quit buttons
  4. When complete, you'll see a dialog: "BrainBlast scenes built successfully!" Click OK.
👁️ What You Should See
In the Project panel, navigate to Assets/Scenes/. You should see two scene files:
  • SplashScreen.unity (sun icon)
  • Gameplay.unity (sun icon)
Double-click Gameplay.unity to open it. In the Hierarchy panel, you'll see a Canvas object with child objects for each panel (MainMenuPanel, LevelSelectPanel, etc.). In the Scene view, you'll see a purple/navy UI layout with buttons and text.

4.2 Configure Build Settings

  1. Go to File → Build Settings (Ctrl+Shift+B).
  2. You'll see a "Scenes In Build" list at the top. If it's empty, you need to add the scenes.
  3. Click "Add Open Scenes" or drag the scene files from the Project panel:
    • SplashScreen — index 0 (this is critical — it must be first!)
    • Gameplay — index 1
    You can drag scenes to reorder them. SplashScreen MUST be at index 0.
  4. Make sure both scenes have their checkboxes enabled (checked).
🚨 Critical
If SplashScreen is not at index 0, the game will start on the wrong screen. The SplashScreen scene's script transitions to the Gameplay scene using SceneManager.LoadScene(1).

4.3 Play From Start

When editing the Gameplay scene, pressing Play would skip the splash screen. The project includes a PlayFromStart utility:

  1. Go to BrainBlast → Play From Start in the menu.
  2. This switches to the SplashScreen scene and enters Play mode, giving you the full game flow.
💡 Quick Test
You can also just open the Gameplay scene and press Play (▶️) to jump straight to the main menu for faster iteration. Use Play From Start when you want to test the complete flow.

Chapter 5 — Understanding the Game

5.1 The Core Mechanic: SumToTen

BrainBlast is a number puzzle game with one simple rule:

Select tiles whose numbers add up to exactly 10.

For example:

  • Tap 7 and 3 → they sum to 10 → tiles clear ✅
  • Tap 2, 3, and 5 → they sum to 10 → tiles clear ✅
  • Tap 6 and 6 → they sum to 12 → invalid, tiles deselect ❌
  • A single 10 tile → tapping it alone clears it ✅

The goal is to clear all tiles from the board in as few moves as possible.

5.2 Worlds & Levels

5.4 Star Rating System

After completing a level, players earn 1–3 stars based on the number of moves used:

StarsCriteria
⭐⭐⭐ (3 stars)Completed within the optimal move count (par or below)
⭐⭐ (2 stars)Completed within 1.5× the optimal move count
⭐ (1 star)Completed the level (any number of moves)

Stars accumulate and can be used to unlock bonus content. Players can replay levels to improve their star rating.

5.5 How levels.json Works

Level definitions are stored in Assets/Resources/LevelData/levels.json. This file is loaded at runtime by LevelManager. Each level entry looks approximately like this:

{
  "levelId": 1,
  "world": "Garden",
  "difficulty": "Easy",
  "gridWidth": 3,
  "gridHeight": 3,
  "tiles": [5, 5, 3, 7, 2, 8, 1, 9, 4],
  "parMoves": 5,
  "starThresholds": [5, 7, 99]
}
  • levelId — Unique number (1–200)
  • world — Which world this level belongs to
  • difficulty — Easy / Medium / Hard / Expert
  • gridWidth / gridHeight — Board dimensions
  • tiles — Array of numbers filling the grid (left-to-right, top-to-bottom)
  • parMoves — The minimum/optimal number of moves to solve
  • starThresholds — Moves needed for [3-star, 2-star, 1-star] (1-star is always 99 = any)

Chapter 7 — Customizing the Game

7.1 Changing Colors & Theme

Open Assets/Scripts/ThemeManager.cs in your code editor. The theme system has two themes: Light (default) and Dark.

Find the ThemeData initialization where colors are defined. Key colors to change:

// Example color properties in ThemeData:
backgroundColor    // Main background color
primaryColor       // Buttons, headers, accents
accentColor        // Secondary highlights
successColor       // Correct match feedback (green)
errorColor         // Invalid match feedback (red)
textColor          // Primary text
secondaryTextColor // Dimmed/subtitle text
cardColor          // Panel/card backgrounds
buttonTextColor    // Text on buttons

Change any HexColor("#XXXXXX") value to your desired color. Unity will apply it to all themed UI elements.

💡 Quick Color Change
The SceneBuilder also defines a color palette at the top of Assets/Editor/SceneBuilder.cs. If you want to change the initial build colors, edit these values before running Build All Scenes: 
private static readonly Color DarkBg       = HexColor("#1A1A2E");  // Background
private static readonly Color Purple       = HexColor("#7C3AED");  // Primary
private static readonly Color YellowGold   = HexColor("#F1C40F");  // Accents
private static readonly Color Green        = HexColor("#2ECC71");  // Success
private static readonly Color Red          = HexColor("#E94560");  // Error

7.2 Adding New Levels

  1. Open Assets/Resources/LevelData/levels.json in any text editor.
  2. Add a new entry at the end of the JSON array following the existing format: 
    {
  "levelId": 201,
  "world": "Crystal",
  "difficulty": "Expert",
  "gridWidth": 5,
  "gridHeight": 5,
  "tiles": [3, 7, 2, 8, 1, 9, 4, 6, 5, 5, 3, 7, 2, 8, 1, 9, 4, 6, 5, 5, 1, 9, 2, 8, 3],
  "parMoves": 12,
  "starThresholds": [12, 16, 99]
}
  3. Important rule: Every tile arrangement must be solvable! Ensure that all tiles CAN be grouped into sets summing to 10. The total of all tile numbers must be divisible into groups of 10.
  4. Save the file and return to Unity. It will auto-reimport the JSON.
⚠️ Validation
There's no built-in level validator yet. Test each new level by playing it to make sure it's solvable. Alternatively, use ProceduralLevelGenerator.cs which generates guaranteed-solvable boards.

7.3 Changing the Mascot

  1. Create or obtain your mascot sprite (PNG with transparency, recommended 512×512 pixels).
  2. Drag it into Assets/Sprites/Mascot/ in the Unity Project panel.
  3. Select the imported sprite. In the Inspector, set:
    • Texture Type: Sprite (2D and UI)
    • Pixels Per Unit: 100
    • Click Apply
  4. Open the Gameplay scene, find the mascot Image component in the Hierarchy (under the relevant panel), and drag your new sprite into its "Source Image" field in the Inspector.

7.4 Modifying Tile Colors

Tile colors are defined in BoardRenderer.cs. Look for the color array or dictionary that maps number values to colors. Change the Color values to your preference:

// Example (colors may vary in actual code):
Color tileColor1 = HexColor("#FF6B6B"); // Number 1 - Red
Color tileColor2 = HexColor("#4ECDC4"); // Number 2 - Teal
Color tileColor3 = HexColor("#45B7D1"); // Number 3 - Blue
// ... etc

7.5 Changing the Game Title

  1. In-game title: Open SceneBuilder.cs and search for "BrainBlast" — replace it with your desired title. Rebuild scenes.
  2. App name (on device): Go to Edit → Project Settings → Player. Change "Product Name" to your desired name.
  3. Splash screen text: Edit SplashScreen.cs or the SplashScreen scene's text objects.

Chapter 8 — Adding Google AdMob (Optional)

⚠️ This is Optional
The game works perfectly without ads. Only add this if you want to monetize with advertisements. The ad code is already written in AdManager.cs — you just need to enable it.

8.1 Download the Plugin

  1. Go to Google Mobile Ads Unity Plugin Releases.
  2. Download the latest .unitypackage file (e.g., GoogleMobileAds-v9.x.x.unitypackage).

8.2 Import into Unity

  1. In Unity, go to Assets → Import Package → Custom Package…
  2. Navigate to the downloaded .unitypackage file and open it.
  3. An import dialog appears showing all files. Click "Import" (import everything).
  4. Wait for compilation. The Google Mobile Ads plugin adds several folders under Assets/.

8.3 Enable the AdManager

  1. Go to Edit → Project Settings → Player.
  2. Expand "Other Settings".
  3. Find "Scripting Define Symbols" (under the "Script Compilation" section).
  4. Add GOOGLE_MOBILE_ADS to the list. If there are existing symbols, separate with a semicolon: 
    GOOGLE_MOBILE_ADS
  5. Press Enter or click Apply. Unity will recompile scripts.
👁️ What You Should See
After adding the define symbol, Unity recompiles. The Console should show no new errors. The AdManager.cs code inside #if GOOGLE_MOBILE_ADS blocks is now active.

8.4 Configure Ad Unit IDs

  1. Open Assets/Scripts/AdManager.cs in your code editor.
  2. Find the ad unit ID strings. Replace them with your own from the AdMob dashboard: 
    // Replace these with your real ad unit IDs:
private string rewardedAdId = "ca-app-pub-XXXXX/YYYYY";
private string interstitialAdId = "ca-app-pub-XXXXX/ZZZZZ";
  3. For testing, use Google's official test IDs (already in the code or use): 
    // Android test IDs:
Rewarded:      "ca-app-pub-3940256099942544/5224354917"
Interstitial:  "ca-app-pub-3940256099942544/1033173712"
🚨 Never Test With Real Ad IDs
Using real ad unit IDs during development can get your AdMob account banned. Always use test IDs during development and only switch to real IDs for production builds.

8.5 Google AdMob App Setup

  1. Go to Assets → Google Mobile Ads → Settings (this menu appears after importing the plugin).
  2. Enter your AdMob App ID (found in your AdMob dashboard under App Settings). It looks like: ca-app-pub-XXXXXXXXXXXXXXXX~YYYYYYYYYY.
  3. Enable the platforms you're targeting (Android and/or iOS).

9.4 Keystore Setup (Signing)

Android apps must be digitally signed. You need to create a keystore (once) and use it for all builds.

  1. In Player Settings, scroll to "Publishing Settings".
  2. Check "Custom Keystore".
  3. Click "Keystore Manager" → "Create New".
  4. Choose a location to save the keystore file (e.g., brainblast.keystore). Choose a strong password.
  5. Fill in the key details:
    • Alias: brainblast
    • Password: (choose a strong password)
    • Validity (years): 25
    • Fill in at least the Organization field
  6. Click Add Key / Create.
🚨 NEVER Lose Your Keystore!
Back up your .keystore file and remember your passwords! If you lose the keystore, you cannot update your app on Google Play — you'll have to create a completely new app listing. Store the keystore and passwords in a safe place (password manager, encrypted backup).

9.5 Build the APK / AAB

  1. Go to File → Build Settings.
  2. For testing on your device, click "Build" to create an APK file. Choose a filename like BrainBlast.apk.
  3. For Google Play, check "Build App Bundle (Google Play)" checkbox, then click Build. This creates an AAB file.
  4. Wait for the build to complete. First-time IL2CPP builds can take 10–30 minutes.
💡 APK vs AAB
  • APK — Direct install on devices. Use for testing.
  • AAB (Android App Bundle) — Required by Google Play. Google generates optimized APKs for each device configuration.

9.6 Testing on a Physical Device

  1. On your Android phone: Go to Settings → About Phone → tap "Build Number" 7 times to enable Developer Options.
  2. Go to Settings → Developer Options → enable USB Debugging.
  3. Connect your phone via USB cable. When prompted on the phone, tap "Allow USB Debugging".
  4. Install the APK via terminal: 
    adb install BrainBlast.apk
    Or in Unity, use "Build and Run" which installs and launches automatically.
👁️ What You Should See
The app icon appears on your phone's home screen/app drawer. Tapping it launches the splash screen followed by the main menu. Test all game flows: play a level, check settings, verify audio works.

11.2 Resetting the Project

If things get really broken, you can reset Unity's cached data:

  1. Close Unity completely.
  2. Navigate to the project folder in your file explorer.
  3. Delete the Library/ folder. (This is safe — it's just a cache.)
  4. Optionally delete the Temp/ folder too.
  5. Reopen the project in Unity. It will reimport everything from scratch (takes a few minutes).
💡 When to Reset
Reset when: sprites appear pink, scripts don't recompile even after fixing errors, weird behavior that shouldn't happen. Do NOT delete: Assets/, Packages/, ProjectSettings/ — those contain your actual project data.

11.3 Reading the Console

Pro tip: Click the three toggle buttons at the top of the Console to filter by type. When debugging, focus on red errors first.

11.4 "Firebase not configured" Warning

You may see a warning about Firebase not being configured. This is expected and harmless. The Analytics system has optional Firebase integration. If you don't need Firebase analytics, simply ignore this warning. The game works perfectly without it.
